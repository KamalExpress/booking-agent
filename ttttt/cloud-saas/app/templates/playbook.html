Slot Monitor Playbook

End-to-end guide on setting up your distributed worker fleet to monitor the portal.

1

Add Proxies

Workers need rotating or sticky proxies to bypass Web Application Firewalls (WAFs) and avoid IP bans.

  • Navigate to Execution Fleet > Proxies.
  • Add your proxy string (e.g. http://user:pass@host:port).
  • The system will test the proxy. If successful, it becomes READY.
2

Add Portal Accounts

You must provide credentials that the workers can use to authenticate with the target portal.

  • Navigate to Execution Fleet > Portal Accounts.
  • Input the Username and Password for the portal.
  • Accounts are automatically rotated by the scheduler to prevent lockouts.
3

Deploy Workers

Workers are headless browser instances that execute tasks. You must run them on your own infrastructure.

  • Start the Python headless worker script on a VPS or local machine.
  • Navigate to Execution Fleet > Workers to verify they are connected.
  • They should appear with an IDLE status, waiting for tasks.
4

Create Assignments (Scrapers)

Assignments instruct the scheduler on what to monitor.

  • Navigate to Workloads > Scrapers (Tasks) and create a new Assignment.
  • Specify the Target URL, location, and polling interval.
  • The scheduler will immediately match a Worker + Proxy + Account and dispatch the job!

What happens next?

Auto-Booking

When a Scraper finds a slot, it automatically generates a high-priority Booking Task. An available Worker immediately executes this task to secure the slot.

Monitoring

Use the Event Logs to watch system activity in real-time. If an account or proxy gets rate-limited, it will automatically enter COOLDOWN and recover later.
